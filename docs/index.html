⚙️ Рекомендуемые параметры для стабильной работы

Настройки DNS, TUN-стека и MTU критически важны для производительности и безопасности

🔒 DoH / DoT (DNS over HTTPS / TLS)
Шифрование DNS-запросов для защиты от слежки провайдера и подмены DNS-ответов. Обычный DNS (порт 53) передаётся в открытом виде — любой может видеть какие сайты вы посещаете.
Удаленный DNS (через прокси) ⭐ Рекомендуется
https://security.cloudflare-dns.com/dns-query
Cloudflare — быстрый и надёжный DoH-сервер. Работает через прокси-туннель.
DNS для прямых запросов (локальный трафик)
udp://223.5.5.5
AliDNS — стабильный и быстрый DNS от Alibaba. Используется для запросов, идущих напрямую, минуя прокси.
⚠️ Важно: Не используйте один DNS для прямых и удалённых запросов — это может вызвать DNS-утечки.
🖧 Stack (TUN-режим)
Стек сетевого интерфейса, который обрабатывает весь трафик в TUN-режиме. От выбора стека зависит стабильность соединения и потребление ресурсов.
Рекомендуемый стек ⭐ Лучший выбор
gVisor
Изолированное окружение с высокой безопасностью и стабильностью. Рекомендуется для большинства пользователей.
Альтернативные варианты
Mixed / System
Mixed — гибридный режим (хорош для Windows)
📦 MTU (Maximum Transmission Unit)
Максимальный размер передаваемого пакета. Этот параметр критически важен для протокола Hysteria2, а также для стабильности VPN-соединения в целом.
Рекомендуемое значение ⭐ Оптимально
1280
Универсальное значение для большинства сетей (Wi-Fi, 4G/5G, проводной интернет). Избегает фрагментации пакетов.
Альтернативные значения
1400 (для стабильных сетей) / 1500 (только провод)
Если соединение обрывается — уменьшайте MTU до 1280 или ниже (например, 1200).
💡 Совет для Hysteria2: Установите MTU = 1280 — это гарантирует отсутствие фрагментации и максимальную производительность.
⏱️ Автообновление подписки
Автоматическое обновление списка конфигураций. Серверные конфигурации периодически меняются, без обновления вы можете использовать нерабочие сервера.
Рекомендуемый интервал ⭐ Оптимально
2 часа (120 минут)
Достаточно часто, чтобы всегда иметь актуальные конфиги, но без лишней нагрузки на сервер.
Альтернативные интервалы
4-6 часов (240-360 минут)
Для экономии трафика, если серверные конфиги обновляются редко.
⚠️ Важно: При первом импорте или если подписка недоступна через прокси, используйте функцию «Обновить без прокси».
